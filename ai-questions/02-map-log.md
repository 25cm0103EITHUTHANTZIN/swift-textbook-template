## 質問と回答の記録

### Q1

**質問：**

IDとUUIDの違いはなんですか。

**AIの回答の要点：**

ID は、「データを区別するための番号・識別子」という“役割”の名前です。

例えば：　学生番号　社員番号　商品番号　みたいなものです。

Identifiable　を使うと、「このデータはIDを持っています」という意味になります。

UUID は、「絶対に重複しにくい特別なIDを自動生成する仕組み」です。

UUID()を作ると：A4B8E3F2-1D3C-4B7E-9A2C-123456789ABC　みたいな長い文字になります。毎回違う値が生成されます。

**自分の理解：**

let id = UUID() これは：id → 識別子の名前 / UUID() → ランダムで重複しにくいIDを作る という意味になります。

ID「名札を付ける」id = ? まだ中身は未定。UUID「自動で超特殊な名札番号を作る機械」550E8400-E29B-41D4-A716-446655440000 を生成する。

### Q2

**質問：**

enum　はなんですか。

**AIの回答の要点：**

「決められた選択肢だけを使うための型」です。

例えば信号。

信号は： 赤 青 黄 しかありません。

enumで表すと
```swift
enum Signal {
    case red
    case blue
    case yellow
}
```

**自分の理解：**

enumとは決められた選択肢だけ使う仕組み

### Q3

**質問：**

enum Category: String, CaseIterable　で　String, CaseIterable　とは？

**AIの回答の要点：**

「Categoryに追加機能を付けています」という意味です。String機能　+　CaseIterable機能　を追加している。

**自分の理解：**

enum は、決められた選択肢だけを使うための型であり、このコードでは観光地のカテゴリを安全に管理するために使われている。

Category に String を付けることで、「寺社」などの文字列を持てるようになり、rawValue を使って画面に表示できることを理解した。

また、CaseIterable を付けることで allCases が使えるようになり、カテゴリ一覧を自動で取得して、フィルターボタンを簡単に生成できることが分かった。

さらに、switch self は「今のカテゴリ自身」を判定して、カテゴリごとにアイコンや色を切り替えるために使われていることを理解した。

### Q4

**質問：**

extension とは？

**AIの回答の要点：**

extensionとは「後から機能を追加する」ものです。

最初：　struct Landmark {　}　を作った。後から：　sampleData　を追加したい。

その時に：　extensionを使う。

static　とは？

**AIの回答の要点：**

staticとは　インスタンスを作らなくても使える　という意味。
